=== 영서 ===

1. 분리
- fixture가 클래스별로 있었으면 좋겠다.
- 그리고 다른 fixture에서는 (객제 자체에 의존할지라도) 그 fixture의 변수 이름으로 의존했으면 좋겠다.
  >> 변경을 최소화하기 위해 (어떤 변수의 값이 바뀌더라도, 그것을 사용하는 다른 코드에는 영향이 없을것이니)
- 그러려면 생성의 순서가 정해져있어야 한다.. (하지만 다른 관점에서 보면, 이것은 FK에 의해 당연하다고 생각될 수도 있다.)

2. 테스트 시간
- DatabaseCleaner를 사용해서 데이터베이스를 초기화할 때, 미리 데이터들을 고정해두면 좋을 것 같다,
- 그런데 모든 테스트 함수마다, 모든 데이터베이스를 truncate 하고 다시 모든 데이터를 다 삽입하고.. 그건 좀 아닌 것 같은데
어떤 시점의 데이터들만 원상복귀하면 안되나?
> 지피티한테 물어봤는데 이건 불가능 / 차라리 data.sql을 쓸 바에는 객체로 영속화를 하는게 좋을 듯

3. 테스트 코드 자체의 단순화
- given 절이 너무 길어지면 안된다. (e.g. 대학교 테스트에서 권역 설정부터 쭉 하는건 아닌듯)
- 테스트 코드에는 테스트에 필요한 모든 데이터를 보여줘야 한다고는 하지만, 우리 도메인의 가장 기본적인 것이니 바로 눈에 보이지 않아도 된다 생각한다.
- 생각나는 방법 : 상속, public static -> 그런데 이렇게 하더라도 확실한 테스트 분리를 하려면 전체를 다 삽입하고, 삭제하는 방식을 써야한다..
